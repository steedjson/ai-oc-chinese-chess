#### FE-NEW-001: AI 对战 API 端点错误

**优先级**: 🔴 P0  
**所在页面**: AI 对战 (/ai-game)  
**问题类型**: API 集成  
**发现时间**: 2026-03-06 02:13

**问题描述**:
前端代码调用错误的 API 端点创建 AI 游戏。前端调用 `POST /api/v1/games/ai/`，但后端实际端点是 `POST /api/v1/ai/games/`。导致点击"开始游戏"按钮后显示"服务器无响应"错误。

**影响范围**:
- AI 对战功能完全不可用
- 无法创建游戏房间
- 无法进行人机对弈

**复现步骤**:
1. 登录成功后访问首页
2. 点击"AI 对战"按钮
3. 选择难度等级（如 5）
4. 点击"开始游戏"按钮
5. 观察错误提示："服务器无响应，请稍后重试"

**期望行为**:
- 成功创建 AI 游戏
- 跳转到游戏对弈页面
- 棋盘加载完成，可以开始走棋

**实际行为**:
- 显示错误提示
- 停留在 AI 对战页面
- 无法进行游戏

**技术细节**:
- 前端调用：`POST /api/v1/games/ai/`
- 后端实际：`POST /api/v1/ai/games/`
- 后端响应：405 Method Not Allowed

**建议修复方案**:
1. 定位前端 API 调用代码（可能在 `src/services/api.ts` 或 `src/api/game.ts`）
2. 修改端点路径：`/api/v1/games/ai/` → `/api/v1/ai/games/`
3. 测试验证：创建新游戏并确认成功

**修复负责人**: 前端工程师  
**预计修复时间**: 15 分钟  
**状态**: ✅ 已验证 - 代码已正确，无需修复（2026-03-06 06:55 验证）

#### FE-NEW-005: WebSocket 服务未启动

**优先级**: 🔴 P0  
**所在页面**: 匹配对战 (/matchmaking)  
**问题类型**: 服务依赖  
**发现时间**: 2026-03-06 02:14

**问题描述**:
后端 WebSocket 服务（Django Channels）未启动，导致匹配对战功能无法使用。前端尝试连接 WebSocket 但失败。

**影响范围**:
- 在线匹配功能完全不可用
- 无法进行玩家对战
- 无法实时同步游戏状态

**复现步骤**:
1. 访问匹配对战页面 `/matchmaking`
2. 点击"开始匹配"按钮
3. 观察控制台错误（WebSocket 连接失败）

**期望行为**:
- WebSocket 连接成功
- 进入匹配队列
- 显示匹配进度
- 找到对手后跳转到游戏页面

**实际行为**:
- WebSocket 连接失败
- 无法进入匹配流程

**技术细节**:
- WebSocket 端点：`ws://localhost:8001/ws/matchmaking/`
- 所需服务：Django Channels + Daphne + Redis
- 当前状态：服务未启动

**建议修复方案**:
1. 安装 Django Channels 和 Daphne（如未安装）：
   ```bash
   pip install channels daphne channels-redis
   ```
2. 配置 ASGI（`asgi.py`）：
   ```python
   from channels.routing import ProtocolTypeRouter
   application = ProtocolTypeRouter({...})
   ```
3. 启动 Daphne 服务：
   ```bash
   daphne -b 0.0.0.0 -p 8001 chinese_chess.asgi:application
   ```
4. 启动 Redis（用于频道层）：
   ```bash
   redis-server
   ```
5. 验证 WebSocket 连接

**修复负责人**: 后端工程师  
**预计修复时间**: 30-60 分钟  
**状态**: 待修复
